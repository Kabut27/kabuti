📋 Cloudflare Workers Proxy Setup (Recommended)

Because browsers block direct cross-origin API requests, deploy a small Cloudflare Worker as a proxy alongside this Pages site. Create a new Worker with the code below: 

/* Cloudflare Worker – proxy.js
   Deploy this Worker separately, then set PANEL_PROXY_URL
   in this page to your Worker URL instead of the raw panel URL.
*/

export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    const panelBase = env.PANEL_URL; // Set this in Worker env vars
    const targetUrl = panelBase + url.pathname + url.search;

    const proxyRequest = new Request(targetUrl, {
      method: request.method,
      headers: {
        ...Object.fromEntries(request.headers),
        'host': new URL(panelBase).host,
      },
      body: request.method !== 'GET' ? request.body : undefined,
    });

    const response = await fetch(proxyRequest);
    const newResponse = new Response(response.body, response);
    newResponse.headers.set('Access-Control-Allow-Origin', '*');
    newResponse.headers.set('Access-Control-Allow-Headers', 'Authorization, Content-Type');
    return newResponse;
  }
};

Set PANEL_URL as a Worker secret (your raw panel URL), then use the Worker URL in the login form above.
